# Redirect Manager

Панель управления ссылками и QR-кодами для печатных изданий.
<img width="1154" height="554" alt="Screenshot_25" src="https://github.com/user-attachments/assets/ec54569b-bda2-45de-b91f-57e7fb93bc19" />


## Описание

**Redirect Manager** — это веб-приложение (Node.js + Express), которое позволяет гибко управлять редиректами и генерировать QR-коды для печатных книг, брошюр, рекламных материалов. Вместо прямых URL‑адресов вы печатаете ссылки на ваш сайт (например, `https://ваш-сайт/redirect/{id}`), а в панели управления можете в любой момент изменить конечный URL или подставить заглушку, если ресурс временно недоступен. Это избавляет от необходимости переиздавать книгу.

**Ключевая идея** — вы управляете переадресацией централизованно, без участия программистов и без переиздания материалов.

## Основные возможности

### 1. Авторизация и роли
- **Три уровня доступа**:
  - **Администратор** — полный доступ ко всем функциям (управление ссылками, тегами, пользователями, настройками, шаблонами, бэкапами, смена паролей других пользователей).
  - **Пользователь** — управление тегами и ссылками, редактирование заглушек, смена собственного пароля. Без доступа к настройкам, пользователям, шаблонам писем и бэкапам.
  - **Гость** — только просмотр (без возможности изменений).
- **Хеширование паролей** (bcrypt) — безопасное хранение.
- **Смена пароля** через интерфейс (для текущего пользователя и администратором для других).

### 2. Управление тегами
- **Группировка ссылок** по темам (например, «Акции 2025», «Документация», «Контакты»).
- Для каждого тега указывается **email для уведомлений** о недоступности ссылок.
- Визуальный **индикатор** — если в теге есть недоступные ссылки, рядом с названием появляется красный значок с числом проблемных ссылок.
- CRUD (создание, редактирование, удаление) тегов.

### 3. Управление ссылками
- **Добавление, редактирование, удаление** ссылок.
- Поля: имя (для идентификации), URL (конечный адрес), привязка к тегу.
- **QR-код** генерируется автоматически для каждой ссылки. Встроенный **редактор стилей** позволяет менять цвет точек, цвет фона, форму точек, добавлять прозрачный фон и загружать логотип. Скачивание QR-кода в PNG.
- **Статус доступности** — «Доступен» / «Недоступен» (обновляется при проверке). Для медленно загружаемых сайтов настраивается таймаут проверки.
- **Кэширование файлов** — если ссылка ведёт на файл (PDF, изображение, документ), её можно отметить как «Это файл». Система скачает файл и будет отдавать его локально, даже если оригинал недоступен. При изменении файла на сервере (по размеру/дате) система обновляет локальную копию.
- **Генерация редиректа** — для каждой ссылки создаётся HTML-файл, который перенаправляет на конечный URL (если доступен) или показывает заглушку (если недоступен).

### 4. Проверка доступности
- **Ручная проверка** — кнопка «Проверить» у каждой ссылки.
- **Автоматическая периодическая проверка** — интервал задаётся в настройках (по умолчанию 1 минута для демонстрации, рекомендуем 30–60 минут для продакшена).
- **Прогресс-бар при массовой проверке** — при нажатии «Проверить все ссылки» в кнопке отображается прогресс с количеством обработанных ссылок.
- **Email-уведомления** при обнаружении недоступности:
  - Письма группируются по тегам (одно письмо на тег, а не по каждой ссылке).
  - Получатели: email, указанный в теге, и дополнительно email администратора (из настроек).
  - Повторные уведомления отправляются с заданным интервалом (по умолчанию 24 часа), чтобы не спамить.
  - В письме указывается, используется ли локальная копия файла (если оригинал недоступен).
- **Обновление статуса** и переключение между редиректом и заглушкой.

### 5. Заглушки (страницы при недоступности)
- **Индивидуальная заглушка для каждой ссылки** — редактируется через WYSIWYG-редактор (Jodit) с поддержкой загрузки изображений.
- **Глобальный шаблон заглушки** — используется для новых ссылок (также редактируется через визуальный редактор).
- Заглушка отображается только когда ссылка недоступна; при восстановлении доступности показывается редирект (заглушка при этом остаётся, но не показывается).

### 6. Email-уведомления и шаблоны
- **Настройка SMTP** через интерфейс (хост, порт, TLS, логин, пароль, отправитель).
- **Кнопка «Отправить тестовое письмо»** для проверки настроек.
- **Два редактируемых шаблона**:
  - Уведомление о недоступности ссылок (поддерживаются переменные: `{{tag.name}}`, `{{#each links}}...{{/each}}`).
  - Уведомление о резервной копии (переменная: `{{date}}`).
- **Дублирование писем** администратору (в дополнение к email тега).

### 7. Резервное копирование
- **Автоматическое создание бэкапов** — интервал задаётся в настройках (по умолчанию 24 часа).
- **Глубина хранения** — сколько дней хранить старые бэкапы (по умолчанию 7 дней, старые удаляются автоматически).
- **Бэкапы сохраняются** в папку `/backups` на сервере и отправляются на email администратора в виде ZIP-архива.
- **Ручной бэкап** — кнопка «Бэкап сейчас» создаёт копию немедленно и отправляет на почту.
- Сохраняются: `config.json`, `url.json`, `default.html`.

### 8. Управление пользователями (только администратор)
- **Добавление, удаление пользователей**.
- **Изменение роли** (админ / пользователь / гость) через модальное окно с выпадающим списком.
- **Смена пароля любого пользователя** без ввода старого пароля (удобно, если пользователь забыл пароль).

### 9. Настройки
Все настройки доступны только администратору:
- **Base URL** — ваш домен (используется в письмах и как задел для будущих функций).
- **Интервал проверки ссылок** (минуты).
- **Интервал повторных уведомлений** (часы).
- **Таймаут проверки ссылок** (миллисекунды) — для медленных сайтов.
- **Email администратора** — для дублей уведомлений и получения бэкапов.
- **Интервал бэкапа** (часы).
- **Глубина хранения бэкапов** (дни).
- **SMTP-параметры** (хост, порт, TLS, логин, пароль, отправитель).
- **Игнорировать ошибки TLS/SSL** — для отладки (не рекомендуется в продакшене).

### 10. Справка
- Интерактивная (аккордеон) с развёрнутыми инструкциями для менеджеров и руководителей.
- Вынесена в отдельный файл (`help.html`) для удобства обновления.
- Содержит описание всех функций, ролей, частые вопросы и рекомендации.

### 11. Прочее
- **Экспорт/импорт данных** в JSON (для бэкапа и переноса).
- **Favicon** — иконка сайта.
- **Адаптивный интерфейс** — работает на любых экранах.
- **Группировка кнопок в выпадающих меню** — компактный тулбар.

## Архитектура хранения данных

### Файлы конфигурации и данных
- `config.json` — настройки приложения (SMTP, интервалы, роли пользователей, email-шаблоны, и т.д.).
- `url.json` — теги и ссылки (включая статусы доступности, метаданные файлов).
- `default.html` — глобальный шаблон заглушки.

### Папки на сервере
- `/redirects/{тег}/{id}` — для каждой ссылки создаётся папка с файлами:
  - `redirect.html` — страница редиректа (если доступна).
  - `index.html` — страница-заглушка (если недоступна).
  - `file` — локальная копия файла (если отмечено «Это файл»).
  - `metadata.json` — метаданные файла (размер, дата, etag).
- `/backups` — архивы резервных копий.
- `/uploads` — изображения, загруженные через редакторы.

## Установка и запуск

### Требования к серверу
- **Node.js** версии 20 или выше (рекомендуется 22 LTS).
- **npm** (поставляется с Node.js).
- **ОС**: Linux (Ubuntu/Debian) предпочтительна, но работает на любой системе с Node.js.
- **Домен** (или IP) для доступа к панели.
- **SMTP-сервер** (опционально, для email-уведомлений).
- **nginx** (рекомендуется для production).

### 1. Клонирование репозитория
```
git clone https://github.com/ваш-пользователь/redirect-manager.git
cd redirect-manager
```

### 2. Установка зависимостей
```
npm install
```

### 3. Настройка конфигурации
При первом запуске сервер автоматически создаст `config.json` и `url.json` с дефолтными значениями.  
Отредактируйте `config.json` (обязательно укажите `baseUrl`):

```
{
  "users": {
    "admin": {
      "passwordHash": "$2b$10$...",  // будет сгенерирован автоматически, не меняйте вручную
      "role": "admin"
    }
  },
  "port": 3000,
  "baseUrl": "https://your-domain.com",
  "checkIntervalMinutes": 1,
  "notificationIntervalHours": 24,
  "checkTimeout": 15000,
  "fileDownloadTimeout": 60000,
  "adminEmail": "admin@example.com",
  "backupIntervalHours": 24,
  "backupRetentionDays": 7,
  "smtp": {
    "host": "smtp.your-provider.com",
    "port": 587,
    "secure": false,
    "ignoreTLS": true,
    "auth": {
      "user": "your-email@example.com",
      "pass": "your-password"
    },
    "from": "noreply@example.com"
  },
  "emailTemplates": {
    "unavailable": {
      "subject": "⚠️ Недоступны ссылки в теге {{tag.name}}",
      "body": "<h2>Уважаемый пользователь!</h2><p>Обнаружены недоступные ссылки в теге <strong>{{tag.name}}</strong>:</p><ul>{{#each links}}<li><strong>{{this.name}}</strong> — <a href=\"{{this.url}}\">{{this.url}}</a> (статус: {{this.status}})</li>{{/each}}</ul><p>Пожалуйста, проверьте доступность ресурсов.</p>"
    },
    "backup": {
      "subject": "📦 Резервная копия Redirect Manager за {{date}}",
      "body": "<h2>Резервная копия</h2><p>Создана резервная копия конфигурации и данных за <strong>{{date}}</strong>.</p><p>Файлы приложены к письму.</p>"
    }
  }
}
```

- **`baseUrl`** — обязательно укажите ваш домен (или IP) для генерации корректных ссылок редиректа и QR-кодов.
- **`checkIntervalMinutes`** — интервал автоматической проверки всех ссылок (в минутах).
- **`notificationIntervalHours`** — минимальный интервал между повторными уведомлениями для одной ссылки (в часах).
- **`checkTimeout`** — таймаут проверки ссылок в миллисекундах (по умолчанию 15000).
- **`fileDownloadTimeout`** — таймаут скачивания файла в миллисекундах (по умолчанию 60000).
- **`smtp.ignoreTLS`** — если `true`, отключает проверку SSL-сертификата (полезно для тестирования).

Создайте файл .env
```text
NODE_ENV=production
SESSION_SECRET=someting_very_secret(32)
SMTP_PASS=Password
```

### 4. Запуск вручную (для проверки)
```
node server.js
```
Откройте браузер по адресу `http://ваш-сервер:3000`.  
Логин по умолчанию: `admin`, пароль: `password`.

**Сразу смените пароль!**

### 5. Настройка автозапуска через systemd
Создайте файл службы:
```
sudo nano /etc/systemd/system/redirect-manager.service
```

Содержимое:
```
[Unit]
Description=Redirect Manager Service
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/redirect-manager
ExecStart=/usr/bin/node /var/www/redirect-manager/server.js
Restart=always
RestartSec=10
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

Активируйте и запустите:
```
sudo systemctl daemon-reload
sudo systemctl enable redirect-manager
sudo systemctl start redirect-manager
```

Проверьте статус:
```
sudo systemctl status redirect-manager
```

### 6. Настройка nginx (реверс-прокси)
```
sudo nano /etc/nginx/sites-available/redirect-manager
```

Содержимое:
```
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Активируйте сайт:
```
sudo ln -s /etc/nginx/sites-available/redirect-manager /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 7. Настройка HTTPS (рекомендуется)
Используйте Certbot:
```
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## Использование

1. **Вход** — используйте логин и пароль.
2. **Управление тегами** — создавайте, редактируйте, удаляйте теги. Для каждого тега указывается email для уведомлений.
3. **Управление ссылками** — добавляйте ссылки (имя, URL, тег). Для файлов отметьте чекбокс «Это файл» — система будет скачивать файл и кэшировать его локально.
4. **Проверка ссылок** — кнопка «Проверить» у каждой ссылки запускает проверку доступности. Кнопка «Проверить все ссылки» запускает проверку всех ссылок с отображением прогресса в кнопке.
5. **Статус ссылки** — отображается как «Доступен» или «Недоступен». Для файлов — доступен, если локальная копия существует.
6. **Генерация QR-кода** — кнопка «QR» показывает QR-код, ведущий на `/redirect/{id}`. Можно настроить цвет точек, фона, форму, прозрачность и загрузить логотип. Скачать QR-код в PNG.
7. **Редактирование заглушки** — кнопка «Заглушка» открывает визуальный редактор HTML-страницы, которая показывается, если ссылка недоступна.
8. **Редактирование глобального шаблона** — задаёт внешний вид заглушки для новых ссылок.
9. **Настройки** — позволяют изменить Base URL, интервал проверки, таймаут, интервал повторных уведомлений, настройки бэкапа и SMTP. Кнопка «Отправить тестовое письмо» проверяет SMTP.
10. **Смена пароля** — безопасное обновление пароля администратора.
11. **Управление пользователями** — администратор может добавлять, удалять пользователей, менять их роли и пароли.

```

├── url.json                  # Данные (теги и ссылки)
├── default.html              # Глобальный шаблон заглушки

├── backups/                  # Резервные копии (создаются автоматически)

    └── {id_ссылки или global}/
```

## Безопасность

- Пароль хранится в виде bcrypt-хеша в `config.json`.
- Все API-маршруты защищены сессией (express-session) с `httpOnly` и `secure` (в production).
- Рекомендуется использовать HTTPS в production.
- Для production смените секретный ключ сессии в `server.js` на свой (через переменную окружения `SESSION_SECRET`).
- Добавлен rate limiting для защиты от брутфорса.
- Внедрена Content Security Policy (CSP) для защиты от XSS.
- Валидация URL и имени тега.
